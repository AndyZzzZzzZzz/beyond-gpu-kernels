Slide 1: Title SlideTitle: Beyond the GPU: A Microarchitectural Analysis of LLM Tool-Calling Kernels Presenters: Kunpeng Zhang, Fazal Rehman, TJ Grewal Course: CMPT 450Slide 2: Introduction & MotivationThe Paradigm Shift: LLMs are transitioning from passive text generators to active, autonomous agents utilized in complex engineering workflows.The Misplaced Focus: Historically, optimization efforts within generative AI have overwhelmingly targeted the GPU forward-pass, minimizing Time-To-First-Token (TTFT).The New Bottleneck: As tool-calling expands, the execution bottleneck shifts abruptly from the hardware accelerator back to the host CPU.Slide 3: Architectural DivergenceGPU Hostility: Agentic tool tasks behave more like traditional datacenter Remote Procedure Calls (RPC) than highly parallel machine learning kernels.Scalar by Nature: Operations like parsing JSON, traversing filesystems, and querying databases are inherently scalar.The Hardware Penalty: These tasks require deep recursive logic and complex memory pointer chasing, leading to massive Last-Level Cache (LLC) miss rates and branch mispredictions.Slide 4: Core Research QuestionsQuantification: What is the macro-level "Agentic Tax" (latency penalty) imposed strictly by the LLM orchestration framework? Microarchitectural Profiling: How do specific tool classes map to their defining hardware bottlenecks during multi-step loops? Optimization Viability: Do classical, kernel-level hardware optimizations effectively propagate to end-to-end system latency, or are they absorbed by the orchestration layer? Slide 5: Methodology & System ArchitectureDecoupled Harness: We designed a custom benchmarking suite that strictly isolates the Python/FastAPI orchestrator from compiled, native C++ execution kernels.The Model: Qwen-2.5-7B-Instruct running locally in FP16 with sampling disabled to guarantee deterministic ReAct cycles.Cycle-Accurate Telemetry: Integrated the Linux PMU via PAPI to capture strictly scoped hardware events (Instructions Per Cycle, Branch Mispredictions, L3 Cache Misses) exclusively during CPU execution windows.Slide 6: Tool Kernels & Control BaselineCompute-Bound: An Abstract Syntax Tree (AST) evaluator parsing nested math expressions to stress branch prediction.Memory-Bound: A SQLite tool executing forced full-table scans to intentionally thrash the LLC.I/O-Bound: A recursive filesystem walker stressing OS context switches.The "Speed of Light" Control: A direct pipeline baseline that routes prompts to the C++ kernels using static regular expressions, completely bypassing the GPU and LLM orchestration.Slide 7: Key Finding 1 - The "Agentic Tax"Massive Slowdowns: The Agentic Tax disproportionately punishes highly optimized, lightweight tools.I/O Walk: While the native C++ kernel executes in ~15ms, routing it through the Python LLM orchestrator results in a staggering 69.5x latency slowdown.Compute (AST): Suffers a 14.9x latency penalty.Amdahl's Law: The database lookup (taking over 2.5 billion cycles natively) exhibited a lower 2.4x multiplier, proving the framework penalty is a massive fixed-cost tax rather than a proportional scalar.Slide 8: Key Finding 2 - Context Accumulation (Multi-Step)The ReAct Gauntlet: We evaluated the microarchitectural cost of complex multi-tool workflows.The Cold Start Penalty: At step 1, the Python framework executes ~82 million instructions simply to parse and load JSON schema definitions.The Stateless Trap: Because the LLM is stateless, the framework's instruction volume grows linearly with each successive step, continuously re-serializing an expanding conversation history.Slide 9: Key Finding 3 - The Masking EffectApplying Hardware Optimizations: We successfully applied software prefetching (__builtin_prefetch) to the memory-bound database kernel.The Micro Win: This decoupling of memory latency raised kernel IPC by 14.2% and saved ~66 million core CPU cycles.The System Reality: The end-to-end system speedup remained below 1%.Conclusion: The massive execution bloat introduced by the Python orchestration layer statistically swallows and nullifies kernel-level microarchitectural victories.Slide 10: Conclusion & Future WorkThe Verdict: Current agentic architectures are critically crippled by the orchestration bottleneck.Moving Forward: Hardware-software co-design must shift away from optimizing tool-execution kernels and aggressively target the orchestration layer itself.Proposed Solutions:Transitioning to compiled orchestrators (Rust/C++).Adopting continuous batching inference engines (vLLM).Replacing JSON with standardized in-memory binary protocols like Apache Arrow.Slide 11: Live Demo & Q&A(Optional: 2-3 minutes to run the beyond-gpu-kernels benchmarking harness, showing Terminal 1 running the orchestrator and Terminal 2 executing the stress payloads.)
